========================================================================
             C盘清理工具 - 代码审查发现的 Bug 与优化报告
========================================================================

审查日期：2026-05-26
审查代码文件：
- cleaner_core.py (核心逻辑模块)
- clean_gui.py (GUI 界面模块)

以下是审查中发现的主要 Bug 和体验瑕疵，请参考（此文件仅作记录，未对源代码进行任何修改）：

------------------------------------------------------------------------
1. 冗余的控制台黑框闪烁 (Bug/体验瑕疵)
------------------------------------------------------------------------
- 所在位置：clean_gui.py 中的 CleanTab._start_clean 函数
- 问题分析：
  GUI界面程序在执行清理任务前，调用了 `subprocess.Popen(['cmd', '/c', 'cls'], creationflags=subprocess.CREATE_NEW_CONSOLE)`。
  这会导致用户点击“清理”按钮时，桌面突兀地闪现一个黑色 CMD 命令行窗口并瞬间关闭。此行为容易让用户受到惊吓或误以为程序崩溃报错，且对程序实际清理功能毫无作用。
- 修复建议：
  直接将这两行调用 `cmd /c cls` 的 subprocess 代码删除即可。

------------------------------------------------------------------------
2. DNS 刷新命令弹出黑窗 (Bug/体验瑕疵)
------------------------------------------------------------------------
- 所在位置：cleaner_core.py 中的 clean_category 函数
- 问题分析：
  在处理特殊命令（DNS 刷新）时，通过 `subprocess.run(["ipconfig", "/flushdns"], ...)` 默默调用系统命令，但是漏掉了 `creationflags` 隐藏窗口参数。这会导致每当清理含有 DNS 缓存时，也会在桌面闪过一个黑色命令行执行窗口。
- 修复建议：
  在 `subprocess.run` 调用中追加参数 `creationflags=0x08000000` (即 subprocess.CREATE_NO_WINDOW)，从而实现完全静默后台刷新。

------------------------------------------------------------------------
3. 单类别文件清理进度僵死 (Bug/逻辑缺陷)
------------------------------------------------------------------------
- 所在位置：clean_gui.py 中的 CleanTab._run_clean 函数
- 问题分析：
  该函数在循环调用 `core.clean_category(result)` 执行真正的底层清理动作时，并没有传入 `progress_callback`。由于像浏览器缓存这样的分类动辄包含数万个小文件，UI 进度条在这个期间无法更新（会停在例如：`正在清理: Chrome 浏览器缓存... (1/X)`），给用户造成“卡死”的错觉。
- 修复建议：
  调用 `core.clean_category` 时传入 Lambda 表达式，关联至界面的 `_on_clean_progress` 以实现单文件级别的进度实时滚动。

------------------------------------------------------------------------
4. 提权路径的潜在安全隐患 (Bug/鲁棒性)
------------------------------------------------------------------------
- 所在位置：cleaner_core.py 中的 run_as_admin 函数
- 问题分析：
  在请求 UAC 管理员提权时使用了相对路径 `script = sys.argv[0]`。如果外部程序从其他目录（非项目根目录）通过命令行拉起这个工具，提权重启时操作系统会因找不到对应脚本而直接退出程序。
- 修复建议：
  改为绝对路径获取：`script = os.path.abspath(sys.argv[0])`，保证跨目录调用环境依然健壮。
